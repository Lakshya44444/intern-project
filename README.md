# FreightIQ Elite v8.6

AI-driven logistics pricing engine — two-stage XGBoost stack predicts delay risk, then converts it into a 5-layer optimal freight quote in real time.

---

## Architecture

```
Browser (React + Vite)
  │
  ├─ AuthPage        →  Supabase Auth (email/password)
  ├─ InputForm       →  City autocomplete via Photon API (OpenStreetMap CDN)
  └─ ResultDashboard →  5-layer pricing breakdown
           │
           │  HTTPS + JWT Bearer
           ▼
FastAPI Backend (app.py)                            uvicorn port 5000
  ├─ /api/autocomplete    →  Photon CDN proxy + server-side LRU cache
  ├─ /api/predict         →  Two-stage XGBoost + 5-layer pricing engine
  └─ /api/extract-vision  →  Claude Vision (invoice / diesel board OCR)
           │
           ├─ Stage 1: XGBoost Classifier  →  P(delay)          AUC 0.91
           └─ Stage 2: XGBoost Regressor   →  efficiency score   R² 0.93
```

| Layer | Technology |
|---|---|

## Setup

### Prerequisites

- Python 3.10+
- Node.js 18+
- A [Supabase](https://supabase.com) project (free tier works)

### 1 — Train models (one-time)

Downloads the Kaggle dataset and produces trained `.pkl` files:

```bash
cd final_intern
pip install -r requirements.txt

python freightiq_01_data_processor.py
python freightiq_02_feature_engineer.py
python freightiq_03_elite_stack_v8.py
```

Expected output from script 03:
```
AUC  = 0.91  |  Brier = 0.09
R²   = 0.93  |  RMSE  = 0.038
Models saved → models/
```

### 2 — Frontend

```bash
cd frontend
npm install
cp .env.example .env
```

Edit `frontend/.env`:

```
VITE_SUPABASE_URL=https://your-project-ref.supabase.co
VITE_SUPABASE_ANON_KEY=your_anon_public_key_here
```

Get both values: **Supabase Dashboard → Project Settings → API**

## Running

Open two terminals:

```bash
# Terminal 1 — Backend (port 5000)
cd final_intern
python app.py

# Terminal 2 — Frontend (port 5173)
cd final_intern/frontend
npm run dev
```

Open `http://localhost:5173` in the browser.

### Optional backend environment variables

```bash
# Enable JWT verification (required in production)
export SUPABASE_JWT_SECRET=your_jwt_secret   # Dashboard → Settings → API → JWT Secret

# Enable Claude Vision invoice extraction
export ANTHROPIC_API_KEY=your_key
```

Without `SUPABASE_JWT_SECRET` the backend runs in **dev bypass mode** — `/api/predict` accepts all requests without a token. Safe for local development; never deploy this way.

## Authentication Flow

1. User registers or signs in on `AuthPage` via the Supabase JS SDK
2. Supabase issues a JWT `access_token` on success
3. `App.jsx` attaches it as `Authorization: Bearer <token>` on every `/api/predict` call
4. Backend `verify_token()` validates the JWT with `SUPABASE_JWT_SECRET` (HS256)
5. On 401 response the frontend auto-signs out and returns to the login screen

### `POST /api/predict`

**Auth:** `Authorization: Bearer <supabase_access_token>` (bypassed in dev mode)

**Request body**

```json
{
  "origin":             "Delhi",
  "destination":        "Mumbai",
  "weight_tons":        12.5,
  "vehicle_class":      "Medium",
  "shipment_date":      "2026-05-01",
  "add_ons":            ["GPS Tracking", "Insurance"],
  "origin_coords":      { "lat": 28.6139, "lon": 77.2090 },
  "destination_coords": { "lat": 19.0760, "lon": 72.8777 }
}
```

| Field | Type | Constraints |
|---|---|---|
| `origin` | string | City name |
| `destination` | string | City name |
| `weight_tons` | float | 0 < weight ≤ 50 |
| `vehicle_class` | string | See vehicle classes below |
| `shipment_date` | string | `YYYY-MM-DD` |
| `add_ons` | string[] | Optional services |
| `origin_coords` | object \| null | `{lat, lon}` — from autocomplete |
| `destination_coords` | object \| null | `{lat, lon}` — from autocomplete |

**Vehicle classes:** `Mini` · `Small` · `Medium` · `Large` · `Extra Large`

**Add-ons:** `GPS Tracking` · `Insurance` · `Express Delivery` · `Fragile Handling` · `Cold Storage`

**Response**

```json
{
  "delay_probability": 0.23,
  "efficiency_score":  0.81,
  "distance_km":       1422,
  "pricing": {
    "operational":   18400,
    "risk_adjusted": 19550,
    "ml_optimized":  20100,

    "final_quote":   19900

  "abstraction": {
    "tier":         "Standard",
    "confidence":   "High",
    "risk_label":   "Low Risk"
  }
}
```

## ML Model Details

### Training pipeline

| Script | Role |
|---|---|
| `freightiq_01_data_processor.py` | Cleans raw dataset, encodes categoricals, creates NaN-safe distance bands (`np.where`) |
| `freightiq_02_feature_engineer.py` | Interaction features, calendar features, lag features |
| `freightiq_03_elite_stack_v8.py` | Trains two XGBoost models via OOF cross-validation |

### Stage 1 — Delay Classifier

- **Target:** binary delay flag
- **Algorithm:** XGBoost (DART booster)
- **AUC:** 0.91 · **Brier:** 0.09

### Stage 2 — Efficiency Regressor

- **Target:** probabilistic efficiency score (0–1)
- **Algorithm:** XGBoost regressor
- **R²:** 0.93 · **RMSE:** 0.038

OOF (out-of-fold) predictions prevent Stage 2 from training on inflated Stage 1 outputs — no data leakage.

## 5-Layer Pricing Engine

| Layer | Name | Description |
|---|---|---|
| 1 | Operational base | `distance × weight × vehicle_rate` |
| 2 | Risk adjustment | `base × (1 + P(delay) × risk_multiplier)` |
| 3 | ML optimization | Corridor + efficiency multiplier from Stage 2 |
| 4 | Competitive snap | Clamps to market corridor (±15%) |
| 5 | Feasibility gate | Enforces minimum viable margin floor |
